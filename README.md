# 摸鱼岛摸鱼插件（Chrome 扩展）

在浏览器中管理**多个账号**，按需定时调用接口、**插件内直连聊天室 WebSocket** 抢红包与辅助自动化；支持**每日签到 / 自动公屏发言**（走 WS）、**宠物抚摸喂养**，并在弹窗中查看、筛选与清空日志。

## 安装与加载

1. 打开 Chrome（或 Edge），地址栏输入 `chrome://extensions`（Edge 为 `edge://extensions`）。
2. 打开右上角 **「开发者模式」**。
3. 点击 **「加载已解压的扩展程序」**，选择本项目的 **`api_plugin` 文件夹**（包含 `manifest.json` 的目录）。
4. 工具栏出现图标后，点击即可打开弹窗。

更新代码后，在扩展管理页对该扩展点击 **「重新加载」**。

## 界面结构

弹窗分为 **「配置」** 与 **「日志」** 两个标签页。配置区内各块为**可折叠区块**（点击标题展开）：账号、定时 API 任务、抢红包与 WS、每日签到与发言、宠物喂养抚摸。

---

## 快速上手（推荐顺序）

1. **账号**：新增或选中账号 → 填 **Token**（可先打开摸鱼岛网页再点「获取页 Token」）→ 若有「未登录」类错误，核对 **Token 请求头名称** 与网页 `localStorage.tokenName` → 需要宠物自动化时再填或获取 **宠物 ID** → 点 **「保存账号资料」**。
2. **全局地址**：在「抢红包与 WS」里确认 **API 基础地址**、**WS 模板**（含 `{token}`）无误 → **保存抢红包与 WS 设置**。
3. **按功能勾选账号**：在「定时 API」「抢红包与 WS」「每日签到与发言」「宠物」各区分别勾选要参与的账号（互不强制一致，可按需组合）。
4. **日志**：执行一段时间后打开「日志」标签 → 用 **查看账号** 过滤 → 必要时 **清空全部日志**。

## 各区块「账号勾选」做什么？

| 区块 | 勾选含义 |
|------|-----------|
| **定时 API 任务 · 执行任务使用** | 到达间隔时，用**每个勾选账号各自的 Token/请求头**对该任务 URL 各请求一次。 |
| **抢红包与 WS · 抢红包/WS 使用** | 为**每个勾选账号**维护一条聊天室 WebSocket；解析到红包 ID 后，由该账号（或网页桥接场景下的勾选列表）发起抢红包 HTTP。**关闭「插件内直连 WS」则不会建立这些连接。** |
| **每日签到与发言 · 批量配置** | 将签到/发言开关、`client` 文案一次性写到勾选账号上；**是否真的会调度闹钟取决于各账号自身已保存的开关**，与「抢红包」勾选无直接关系。 |
| **宠物 · 自动执行任务账号** | 仅影响抚摸/喂养闹钟；需要账号内已保存 **宠物 ID** 且勾选 **开启自动**。 |

网页 **content 脚本** 发现红包但未带账号 ID 时，会对「抢红包」勾选列表里的账号**逐个入队**尝试抢红包（带随机间隔，避免瞬时并发）。

## 配置用法

### 账号（必须先配置）

- **多账号**：通过下拉框切换「编辑资料 · 账号」，可 **新增账号 / 删除**；每条账号独立保存 **备注名**、**Token**、**Token 请求头名称**、**宠物 ID** 等。
- **首次使用**：若你以前只用「单 Token」旧版存储，扩展会自动迁出一条默认账号；之后请以账号为单位保存资料。
- **账号名称**：仅为本地备注，**不会**当作聊天室昵称发送；公屏发言展示名来自服务端 **`GET /api/user/get/login`** 返回的用户信息。
- **Token 请求头名称**：默认 `fish-dog-token`。摸鱼岛网页登录后，Sa-Token 的头名可能为 **`satoken`** 或与 localStorage 里 **`tokenName`** 一致；若签到、发言前拉取用户失败并提示「未登录」，请与浏览器 **Application → Local Storage** 中的 **`tokenName`** 对齐后，再点 **「保存账号资料」**。
- **保存账号资料**：将当前编辑框中的 Token、请求头、备注名、宠物 ID 等写入当前选中账号。
- **获取页 Token**：在已登录摸鱼岛的标签页上点击，会读取该页 `localStorage` 中的 `tokenValue`（及可用的头名信息）填入表单（仍需 **保存账号资料** 才会持久化）。
- **宠物 ID**：可手写，或点 **「获取宠物 ID」**（依赖当前输入框或已保存的 Token）；获取成功后若需长期生效请 **保存账号资料**。自动宠物任务只使用已保存的宠物 ID，不会在运行时再去「查我的宠物」接口。

### 定时 API 任务

- 填写任务名称、完整 API 地址（`http`/`https`）、方法（GET/POST）、间隔（秒），点 **「新增任务」**。
- **执行任务使用（可多选）**：勾选哪些账号参与定时请求；到时后台会对**每个勾选账号各调用一次**该 URL，请求头上附带该账号保存的 **Token 请求头名 + Token**（与摸鱼岛接口约定一致，**不是**固定的 `Authorization: Bearer`）。
- GET 请求通常不带 `Content-Type`；若接口报错请对照浏览器里同类请求的 Header。

### 抢红包与 WS

- **抢红包 / WS 使用（可多选）**：勾选的账号会各自建立一条 **WebSocket**（模板里的 `{token}` 替换为该账号 Token），用于解析红包消息并调用抢红包接口。
- **开启自动抢红包**：总开关；关闭后不发起抢红包 HTTP。
- **插件内直连 WS 抢红包**：勾选后立即生效写入存储；后台按勾选账号连接 WS。
- **聊天室 WS 模板**：例如 `wss://api.yucoder.cn/ws/?token={token}`。
- **抢红包 API 基础地址**：例如 `https://api.yucoder.cn`。抢红包、每日签到、宠物抚摸喂养、拉取登录用户信息等 **REST** 均使用该域名（与摸鱼岛线上 API 一致）。
- 修改 WS 或基础地址后，点 **「保存抢红包与 WS 设置」**。

### 每日签到与发言

- **批量配置每日任务（可多选）**：勾选多个账号后，下方的签到 / 发言开关与「聊天 client」会一次性写入这些账号。
- **每日自动签到**：在本地日期的随机时段调用 **`POST /api/user/signIn`**。
- **每日自动发言 20 条**：内容依次为字符串 **`1`～`20`**，条与条之间有较长随机间隔；**不再调用** `POST /api/chat-room/send`，改为与 uTools 一致的 **WebSocket 帧**（先 `type: 1`，再 `type: 2` 聊天载荷）。发言前会请求 **`GET /api/user/get/login`**，用返回用户组装 `sender`，并对常见 Token 头名做依次尝试；若仍失败，日志中会提示检查请求头与 Token。
- **聊天 client 标识**：历史兼容字段；当前 WS 帧主体与网页端一致时不依赖该项，可留空使用默认值。
- **立即执行签到与发言**：按勾选账号依次立即签到；发言任务在后台异步执行。若该账号 **今日 20 条已完成**，手动按钮会跳过发言并写日志说明。

### 宠物喂养抚摸

- **自动执行任务账号（可多选）** + **自动执行间隔** + **开启自动喂养抚摸**：到达间隔后对勾选账号调用抚摸 / 喂养接口（使用 **抢红包 API 基础地址**）。需已保存 **宠物 ID** 与有效 Token；接口冷却过短时日志可能表现为失败。
- **立即执行喂养抚摸**：按勾选账号执行一轮，不要求已开自动。

### 自动调度时间窗（本地时区）

以下时间为扩展内 **`randomTimeTodayBetween` / `randomTimeTomorrowBetween`** 所用区间，均为**本机本地日历日**，非 UTC。

| 任务 | 调度说明 |
|------|-----------|
| **每日自动签到** | 当天若尚未记为「已签到」：在当日 **07:00～11:45** 内随机一刻触发；若当天已签到：预约 **次日** 同一时间窗内随机一刻。失败会在一段时间后重试闹钟（见接口调用日志）。 |
| **每日自动发言（20 条）** | 当天若 **`dailyLastChatRoutineDay` 已是今日**：预约 **次日** **14:00～22:20** 内随机一刻；若今日未完成且存在未发送序号（断点续跑）：约在 **数秒～十余秒** 后再触发一批；否则首次安排在当日 **14:00～22:20** 内随机一刻。单账号发言循环内部每条间隔约 **22～130 秒**（随机）。 |
| **宠物自动喂养抚摸** | 按所选 **间隔分钟** 排程；手动「立即执行」不强制开启自动，也不会为「仅手动」场景顺延下一次闹钟。 |
| **定时 API 任务** | 使用较短周期的 **chrome.alarms**（约半分钟粒度）轮询到期任务，与上表独立。 |

### WebSocket：抢红包与发言的关系

- **抢红包 WS**：仅在「开启自动抢红包」且「插件内直连 WS」开启，且「抢红包」下有勾选带 Token 账号时保持连接；连接成功后在短随机延迟内发送 **`{ type: 1 }`**（登录帧）。当前扩展未单独实现 `type: 4` 心跳；若长时间掉线可依赖断线重连或仍以网页聊天室保持会话。
- **每日发言**：每条消息走 **临时连接** 或与上述 **同一账号已有 OPEN 连接复用**（优先复用，避免重复建连）。发言帧与 uTools 类似：**先 `type: 1`，再发 `type: 2` 的 chat 载荷**；日志里 **「WS 帧已发出」** 表示浏览器侧 `send` 成功，**不等于**服务端一定写入公屏历史。

## 日志用法

- **清空全部日志**：清除接口调用记录、聊天室 WS 摘要、抢红包明细等本地缓存（不可恢复）。
- **查看账号**：下拉筛选部分日志展示范围。
- **聊天室 WS**：后台记录的 WS 文本摘要（名字：内容；红包 / 图片有特殊标记）；点 **「刷新」** 从存储重新渲染。
- **抢红包 · 按日统计 / 明细**：成功次数、积分与每次调用摘要。
- **接口调用**：定时任务、签到、发言（标记为 WS）、抢红包、WS 连接等摘要列表。
- **接口调用**区域通常较高（`logs-api`），长文案可在条目内滚动查看；红包相关接口日志中的 WS URL 已对 **token** 做脱敏（`token=***`）。

## 抢红包的两种数据来源

1. **插件内直连 WS**（与「抢红包与 WS」中的多选账号配合）：后台为每个勾选账号建立 WS，解析消息中的红包 ID 并入队抢红包。
2. **网页内 WebSocket**：向 `http/https` 页面注入脚本监听页面里的 `WebSocket`；若消息中含 `[redpacket]...[/redpacket]`，向后台上报红包 ID。在摸鱼岛网页打开聊天室即可触发。

抢红包 HTTP 始终使用你在弹窗中配置的 **API 基础地址**，不会误用网页 `origin`。

同一红包 ID 在同一账号下会做 **本地已抢缓存**，避免重复提交；队列内多条任务之间带有 **随机 Sleep**，模拟间隔点击。

## 公屏发言为什么在网页里有时看不到自己那条？

服务端可能对发送者**不推送自己的广播副本**，网页需要自己插入本地消息。插件侧只在后台发 WS，不会像浏览器聊天页那样插入 DOM。可在聊天室 **刷新或拉历史**确认是否入库；换账号登录查看也可区分「未入库」与「自己不显示」。

## 常见问题（排查）

| 现象 | 可检查项 |
|------|-----------|
| 发言日志：**拉取登录用户失败 / 未登录** | Token 是否过期；请求头是否与网页 **`tokenName`** 一致；API 基础地址是否指向当前环境。 |
| 发言：**WS 超时 / 连接错误** | WS 模板是否正确、`{token}` 是否保留；网络是否拦截 **wss**；该账号 Token 是否与其它接口一致。 |
| 定时任务 **401 / 未登录** | 「执行任务使用」里账号是否勾选；该账号是否已 **保存账号资料**。 |
| **抢红包无反应** | 「开启自动抢红包」是否打开；插件 WS 是否勾选账号且 Token 有效；或摸鱼岛网页是否打开聊天室（走页面 WS 桥）。 |
| **宠物一直跳过或失败** | 是否已保存 **宠物 ID**；是否勾选宠物账号并开启自动；接口冷却期内会失败，可调大间隔。 |
| **每日发言只发到一半** | 中途失败会记下 **`dailyChatState` 断点**，闹钟会在短时间内 **续跑**；查看接口调用里对应序号的 WS 摘要。 |

- 若需确认网页侧是否上报红包：在摸鱼岛页面打开控制台，结合 **接口调用** 与 **抢红包明细** 日志对照。

## 权限说明（manifest 摘要）

- **alarms**：定时任务、每日签到与发言批次、宠物间隔等调度。
- **storage**：多账号、任务、日志等。
- **scripting**：从当前标签页读取 `localStorage`（Token）。
- **tabs**：按需访问标签页（如取 Token）。
- **host_permissions**：对配置的 API 与 WS 发起请求；内容脚本注入 `http/https` 页面以监听 WS。

## 注意事项

- Token、请求头名称须与当前摸鱼岛环境一致；勿公开截图或提交 Token。
- 日志与账号数据保存在本机 **chrome.storage.local**；清除浏览器数据或卸载扩展会丢失。
- 若发言日志提示拉取用户失败或 WS 异常，优先核对 **Token 是否过期**、**请求头名称是否与网页 tokenName 一致**、**WS 模板与 API 基础地址**是否正确。
- **自建或换后端域名**时：请同时修改 **REST 基础地址** 与 **WSS 模板**，证书须被浏览器信任；路径需与线上一致（如 `/api/...`、`/ws/`）。
- 自动化功能可能随官方接口或风控策略变化而失效；请合理使用，避免违反站点条款。

## 环境与兼容性

- 面向 **Chrome / Edge** 等 Chromium 浏览器，扩展清单为 **Manifest V3**。
- 后台为 **Service Worker**：可能被浏览器休眠，但 **`chrome.alarms`** 仍可在到期时唤醒以执行定时逻辑；长时间无任何闹钟时，以扩展文档为准。
- **本地时区**：签到 / 发言的「当天」「随机时段」均按 **本机系统时区** 计算；跨时区设备上会不一致，出差时注意校对系统时间。

## 本地数据与隐私（概要）

| 类别 | 说明 |
|------|------|
| **存哪里** | 账号、任务、开关、日志等均在 **`chrome.storage.local`**，默认只在当前浏览器用户配置下；不做云端同步。 |
| **会发往哪里** | 仅向你配置的 **API 基础地址**、**WS 模板**对应主机及你在定时任务里填写的 **URL** 发起请求；不会在扩展作者服务器中转。 |
| **网页脚本** | `content.js` 在所有 `http/https` 页注入（清单权限），用于挂钩 `WebSocket`；若介意广度，可 fork 后收窄 `manifest.json` 里的 `matches`。 |
| **获取页 Token** | 仅在点击按钮时通过脚本读取**当前标签页** `localStorage`（典型为摸鱼岛域名）；必须先打开对应站点标签页再点击按钮，扩展无法静默读取任意站点。 |

日志条目数量等有上限（见下节），超出部分会丢弃最旧记录，避免占满存储配额。

## 定时 API 任务的局限

- 请求 **仅包含方法 + URL + 账号 Token 相关 Header**；当前实现 **POST 不带 body**。若接口要求 JSON body，请改用浏览器 / Postman，或后续在扩展内增强该功能。
- **GET** 未强制添加 `Content-Type`；与摸鱼岛自带 fetch 行为对齐时通常更稳妥。
- **401 / CORS**：扩展后台 `fetch` 受浏览器扩展规则约束；若目标 API 未允许扩展来源或鉴权失败，需在日志里根据状态码与响应正文排查。

## 与其它客户端（网页 / uTools）同时使用

- **同一账号 Token** 可被网页、uTools、本插件同时使用；服务端若有「单端登录踢下线」策略，则一端失效会导致另一端 **`get/login` 失败或 WS 断开**。
- **发言协议**：本插件每日发言与 **uTools 聊天室**类似（WS `type: 2` + `data.type: chat`）；服务端升级协议时需同步改扩展。

---

## 卸载与备份

- **卸载扩展**会清除与该扩展关联的存储（除非浏览器另有保留策略）；重要 Token / 配置请先自行抄写或导出（当前版本未提供导出按钮）。
- **删除账号**：在账号下拉选中后点「删除」会从列表移除该账号条目；若误删只能重新添加并粘贴 Token。

## 项目结构（简要）

| 文件 | 作用 |
|------|------|
| `manifest.json` | 扩展清单 |
| `background.js` | 定时任务、多账号 WS、抢红包、签到与 WS 发言、宠物、写日志 |
